Dificuldades Enfrentadas no Módulo 01

Durante o desenvolvimento e a entrega do Módulo 01, enfrentamos algumas dificuldades relacionadas principalmente ao uso do Git e do GitHub.

Inicialmente, criamos o repositório diretamente pelo site do GitHub. Em seguida, desenvolvemos o projeto localmente em nossas máquinas, porém tivemos dificuldades para subir todo o projeto para o repositório utilizando o Git via linha de comando, pois o Git ainda não estava devidamente configurado em nossas máquinas.

Devido a isso, optamos por enviar os arquivos manualmente pelo próprio site do GitHub. Esse processo acabou gerando um problema que só foi percebido posteriormente: algumas pastas do projeto estavam vazias e, por isso, não foram enviadas para o repositório, já que o Git não versiona diretórios vazios. Dessa forma, a estrutura completa de pastas do projeto não apareceu no repositório.

Esse problema só foi identificado durante a preparação da entrega do Módulo 02.

Apesar dessas dificuldades, no Módulo 01 conseguimos cumprir os principais objetivos da atividade, incluindo:

- Entrega da proposta do projeto;
- Criação e disponibilização do repositório no GitHub;
- Execução do projeto Flutter no emulador;
- Criação de um esboço inicial de interface.

No entanto, apenas uma tela foi prototipada inicialmente, pois não havíamos compreendido completamente que era necessário apresentar o esboço de três ou quatro telas principais do aplicativo.

Essas dificuldades contribuíram para nosso aprendizado sobre versionamento com Git, estrutura de repositórios e boas práticas de organização de projetos, o que foi corrigido e melhorado nas etapas seguintes do desenvolvimento.
